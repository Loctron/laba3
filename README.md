# RuleEdtor

## Лабораторная работа №3

### Разработка и исследование редактора правил для продукционной системы

> Дисциплина: **«Представление и использование знаний в интеллектуальных системах»**

---

## Цель работы

Разработать редактор атрибутов экспертной системы, редактор продукционных правил и редактор объяснения результатов вывода, а также приобрести практические навыки представления и использования знаний как компонентов информационных и управляющих систем.

---

## Задачи работы

В ходе выполнения лабораторной работы были поставлены следующие задачи:

* изучить способы представления атрибутов знаний;
* разработать алгоритмы создания и редактирования компонентов базы знаний;
* реализовать пользовательский интерфейс редактора правил;
* разработать форматы представления атрибутов и продукционных правил;
* реализовать поддержку нечётких знаний;
* создать редактор функций принадлежности лингвистических переменных;
* выполнить тестирование и анализ корректности работы системы.

---

# Функциональные возможности

## Работа с проектами

Система поддерживает создание нескольких предметных областей (проектов).

### Возможности:

✔ Создание проекта

✔ Удаление проекта

✔ Переименование проекта

✔ Копирование проекта

✔ Вставка проекта

✔ Иерархия проектов (суперпроекты и субпроекты)

Каждый проект обладает:

| Свойство            | Описание                          |
| ------------------- | --------------------------------- |
| ID                  | Уникальный числовой идентификатор |
| Название            | Смысловой идентификатор проекта   |
| Родительский проект | Поддержка иерархии                |

---

## Работа с атрибутами

Атрибуты используются для описания предметной области и компонентов логического вывода.

### Поддерживаемые типы данных

| Тип     | Описание            |
| ------- | ------------------- |
| NUMERIC | Числовое значение   |
| STRING  | Символьное значение |
| BOOLEAN | Логическое значение |
| DATE    | Дата                |
| FUZZY   | Нечёткое значение   |

### Поддерживаемые типы нечёткости

* Нечёткое число
* Нечёткая переменная
* Лингвистическая переменная

### Операции

* Добавление атрибута
* Удаление атрибута
* Копирование
* Вставка
* Переименование
* Поиск

---

# Поддержка нечёткой логики

Для нечётких атрибутов реализован редактор лингвистических переменных.

## Возможности редактора

* создание предметной шкалы;
* создание терм-множества;
* автоматическая генерация функций принадлежности;
* интерактивное редактирование функций;
* сохранение параметров функций в базе знаний.

---

## Редактор функций принадлежности

Для каждого терма автоматически строится функция принадлежности.

Поддерживаются:

* Треугольные функции принадлежности;
* Трапециевидные функции принадлежности.

Форма функции может изменяться пользователем посредством перемещения управляющих точек на графике.

### Контролируемые ограничения

* упорядоченность точек функции;
* корректность области определения;
* отсутствие пересечений термов;
* полнота покрытия предметной шкалы;
* корректность крайних функций принадлежности.

---

# Редактор продукционных правил

Основное назначение системы — создание базы знаний экспертной системы.

Правила формируются по схеме:

```text
ЕСЛИ
    Атрибут₁ = Значение₁
И
    Атрибут₂ = Значение₂
ТО
    Атрибут₃ = Значение₃
```

### Особенности

* произвольное количество условий;
* выбор атрибутов из словаря проекта;
* выбор значений атрибутов;
* сохранение правил в базе данных;
* поддержка объяснений.

---

# Редактор объяснений

Для каждого правила может быть сформировано пояснение, используемое при интерпретации результатов вывода.

Возможности:

* создание объяснений;
* редактирование;
* сохранение в БД;
* просмотр объяснений для выбранного правила.
